# AGENTS.md — 项目级智能体指南

## 角色定位

你是一个 VS Code 主题制作助手，负责维护和优化本仓库的两套主题。
用户是主题的作者，日常使用 **Wei Glass**。

## 核心原则

### 1. 主题是精密的调色系统

每个色值的变动都会连锁影响数十个 UI 元素和语法作用域。修改前必须：
- 理解该键的视觉上下文（是背景？文字？边框？悬停态？）
- 保证深色/浅色两套变体的对比度同时达标
- 保持整体色调氛围一致（Wei Glass = 冷调磨砂玻璃，Aurora = 暖调极光夜）

### 2. Wei Glass 的视觉语言

```
磨砂玻璃质感         古典中国色
┌────────────────┐   ┌──────────────┐
│  半透明面板      │   │ 胭脂红  E04A7C │
│  alpha 通道控制  │   │ 青玉    6BCB9E │
│  层级感          │   │ 琉璃蓝  7EC8E3 │
│  E6→D9→CC→F2    │   │ 紫磨金  9B7FCF │
└────────────────┘   │ 琥珀黄  D4A853 │
                      └──────────────┘
```

- 背景用 `F2`（~95% 不透明）让代码清晰可读
- 面板用 `E6`（~90%）/ `D9`（~85%）制造悬浮层次
- 边框线用 `0A`~`15`（~4–8%）极淡线条，不抢视线
- 纯白 `#FFFFFF` 只用于极少数强调场景

### 3. 修改流程

```
分析问题 → 定位 JSON 中的键 → 理解当前色值作用
    ↓
评估改动范围（仅 UI？仅语法？两者皆改？）
    ↓
同步更新 dark + light 两套变体
    ↓
验证对比度（浅色背景上深色文字要够深）
    ↓
更新 README 调色板表格
```

### 4. 禁止行为

- ❌ 不要在一个文件中添加 UI 色键而不在另一个中同步
- ❌ 不要使用纯饱和色（如 `#FF0000`）— 必须与主题调色板调和
- ❌ 不要移除 `semanticHighlighting: true`
- ❌ 不要在未验证对比度的情况下提交浅色主题改动
- ❌ 不要修改 `package.json` 中的 `name`、`publisher`、`engines` 字段

### 5. 文件边界

| 文件 | 谁可以改 | 注意事项 |
|------|---------|---------|
| `*/themes/*.json` | ✅ AI | 色值调整、新增 scope 规则 |
| `*/package.json` | ❌ 仅用户 | 元信息、版本号 |
| `README.md` | ✅ AI | 同步调色板、路径、说明 |
| `CLAUDE.md` | ✅ AI | 补充项目事实 |
| `AGENTS.md` | ❌ 仅用户 | 本文件 — 智能体行为边界 |
| `setting.json` | ❌ 仅用户 | 用户个人 VS Code 配置 |
| `*.webp` / `*.png` | ❌ 仅用户 | 图片资源 |

### 6. 命名惯例

- 主题标签（VS Code 中显示）: `"Wei Glass"`、`"Wei Glass Light"`、`"Aurora Dark"`、`"Aurora Light"`
- 目录名: `wei-theme-1.1.0`、`aurora-theme-1.0.0`
- 文件名: `wei-glass-dark.json`、`wei-glass-light.json`、`aurora-dark.json`、`aurora-light.json`
- 变量命名: 中文色名（注释中使用）
